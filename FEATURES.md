# 🎯 YouTube Channel Manager - Advanced Keyword Analysis

## 전체 기능 개요

이 애플리케이션은 YouTube 채널 관리와 포털 키워드 분석을 통합한 올인원 솔루션입니다.

---

## 📋 탭별 기능

### 1️⃣ 📥 Video Downloader
YouTube 채널의 비디오를 대량으로 다운로드합니다.

**주요 기능:**
- 채널 URL/username으로 비디오 목록 조회
- 비디오 정보 표시 (제목, 길이, 조회수, 업로드 날짜)
- 해상도 선택 (1080p, 720p, 480p, 360p)
- 형식 선택 (mp4, mkv)
- 진행 상황 표시 및 배치 다운로드

---

### 2️⃣ 🔍 Keyword Analysis
기본 포털 키워드 분석 기능입니다.

#### 단일 키워드 분석
4개 포털(Google, Naver, Daum, YouTube)의 검색량을 동시 분석합니다.

**분석 항목:**
- 추정 검색량
- 경쟁도 (Google)
- 트렌드 (상승/안정/하강)
- 관련 키워드
- 기회 점수 (Google)
- 검색 의도 (Google)

**포털별 데이터:**
- **Google:** CPC, 난이도 점수, 기회 점수, 월간 검색량
- **Naver:** 관련 키워드, 트렌드, 난이도
- **Daum:** 관련 키워드, 월간 검색량
- **YouTube:** 추정 비디오 수, 추천 검색어

#### 여러 키워드 비교
최대 5개 키워드의 포털별 검색량을 비교합니다.

**비교 항목:**
- 포털별 검색량 (막대 그래프)
- 평균 검색량 (시각화)
- 트렌드 비교
- 평균값 계산

#### 채널 기반 키워드 추천
채널명과 비디오 제목으로부터 자동 키워드 추천합니다.

**추천 방식:**
- 채널명 분석
- 비디오 제목 분석
- 키워드 조합 생성
- 상위 20개 추천

### 3️⃣ 📊 Trend Analysis
30일 트렌드 분석 (Black Kiwi보다 향상된 버전)

**분석 항목:**
- 평균 검색량
- 피크 검색량
- 최소 검색량
- 평균 관심도
- 변동성

**시각화:**
- 30일 검색량 추이 (라인 차트)
- 관심도 추이 (에리어 차트)
- 상세 데이터 테이블

**트렌드 예측:**
- 예측 트렌드 (상승/하강/안정)
- 성장률 (%)
- 신뢰도 (높음/중간/낮음)

**사용자 정의:**
- 분석 기간 선택 (7~90일)

### 4️⃣ 🎯 Advanced Features
고급 분석 기능들입니다.

#### 🔹 숏/롱테일 키워드 분석
키워드를 짧은 키워드(1-2단어)와 긴 키워드(3단어+)로 분류 분석합니다.

**분석 항목:**
- **숏테일 키워드:** 높은 검색량, 높은 경쟁도
- **롱테일 키워드:** 낮은 검색량, 낮은 경쟁도, 높은 전환율

**비교 분석:**
- 검색량 비교
- 난이도 비교
- 추천 (숏테일 vs 롱테일)

**전환율 추정:**
- 상업 키워드 신호 분석
- 전환 가능성 점수 (0-1)

#### 🔹 실시간 추천
다양한 알고리즘으로 최적의 키워드를 자동 추천합니다.

**추천 알고리즘:**
1. **관련 키워드 조합**
   - "best", "how to", "guide", "tutorial" 등 모디파이어 추가

2. **트렌딩 키워드**
   - 현재년도와 함께 조합 (2024, 2025)

3. **니치 키워드**
   - "advanced", "professional", "enterprise" 등 추가

4. **경쟁 낮은 키워드**
   - "specific", "exact", "detailed" 등으로 롱테일화

**점수 계산:**
- 검색량 점수 (30%)
- 경쟁도 점수 (30%)
- 트렌드 점수 (20%)
- 전환율 점수 (20%)

**최종 출력:**
- 점수순 정렬 (상위 20개)
- 타입별 분류 (관련/트렌딩/니치/경쟁낮음)
- 시각화 (막대 그래프)

#### 🔹 경쟁사 분석
경쟁사 키워드 전략을 분석하고 기회를 발굴합니다.

**분석 항목:**
- **공통 키워드:** 경쟁사와 우리 모두가 사용하는 키워드
- **경쟁사 독점:** 경쟁사만 사용하는 키워드
- **우리 독점:** 우리만 사용하는 키워드
- **발견된 기회:** 경쟁사가 사용하지만 우리는 미처리한 고기회 키워드

**기회 점수 계산:**
```
기회 점수 = 검색량 / (난이도 + 1)
```

**필터링:**
- 검색량 > 100 (충분한 수요)
- 난이도 < 50 (취약점 공략 가능)

**시각화:**
- 산점도 (X: 난이도, Y: 검색량, 크기: 기회 점수)
- 상위 10개 기회 키워드 테이블

#### 🔹 검색 의도 분석
키워드의 검색 의도를 4가지 유형으로 분류합니다.

**의도 유형:**

1. **📚 정보 검색 (Informational)**
   - "what", "how", "why", "guide", "tutorial"
   - 사용자가 정보를 찾는 의도

2. **🧭 네비게이션 (Navigational)**
   - "site", "page", "app", "channel"
   - 특정 사이트/앱을 찾는 의도

3. **🛍️ 상업 (Commercial)**
   - "best", "review", "vs", "comparison", "top"
   - 구매 비교/리뷰를 원하는 의도

4. **💳 거래 (Transactional)**
   - "buy", "order", "download", "discount"
   - 구매/다운로드 의도

**분석 방법:**
- 키워드에 포함된 신호 감지
- 신호별 점수 계산
- 주요 의도 판정
- 신뢰도 표시

### 5️⃣ 📈 Performance & History
성능 예측과 히스토리 관리입니다.

#### 🔹 성능 예측
향후 3개월 키워드 성능을 예측합니다.

**예측 방식:**
- 과거 90일 데이터 수집
- 다항식 회귀 분석 (2차)
- 미래 90일 예측
- 신뢰도 계산

**출력 항목:**
- 현재 검색량
- 예측 트렌드 (상승/하강)
- 신뢰도 (%)
- 월별 예측값 시계열 데이터

**그래프:**
- 3개월 검색량 예측 라인 차트

#### 🔹 계절성 감지
연간 키워드 패턴을 분석합니다.

**분석 기간:** 365일

**분석 항목:**
- **월별 패턴:** 12개월 평균 검색량
- **요일별 패턴:** 7일 평균 검색량
- **계절성 강도:** 피크와 저점의 비율
- **피크 월:** 상위 3개월
- **저점 월:** 하위 3개월

**포스팅 일정 추천:**
- 최고의 월
- 최고의 요일
- 피해야 할 월
- 추천 주기 (일일/정기)

**그래프:**
- 월별 검색량 막대 그래프

#### 🔹 키워드 히스토리
과거 분석 결과를 조회합니다.

**데이터 저장:**
- SQLite 데이터베이스
- 포털별 검색량
- 트렌드 정보
- 경쟁도 정보
- 타임스탬프

**조회 기능:**
- 특정 키워드의 과거 분석 조회
- 기간 선택 (1~90일)
- 포털별 분석 데이터

**인기 키워드:**
- Top 10 키워드 자동 표시
- 분석 빈도순 정렬

**시각화:**
- 시간대별 추이 라인 차트 (포털별 색상 구분)

## 📊 주요 분석 알고리즘

### 키워드 난이도 계산
```
기본값: 30
- 길이 > 3단어: -10
- 길이 > 2단어: -5
- 트렌딩 키워드: +15
- 일반 단어 포함: +10
범위: 0-100
```

### 기회 점수 계산
```
기회 점수 = (검색량 / 100) / (난이도 / 50 + 1)
```

### 전환율 추정
```
상업 지표: "buy", "how to", "best", "review", "price"
각 지표당: +0.2
범위: 0-1.0
```

### 계절성 강도
```
강도 = (최대월 - 최소월) / 평균월
```

### 성능 예측
```
방식: 다항식 회귀 (2차)
기간: 과거 90일 -> 미래 90일
신뢰도: 100 - (표준오차 / 평균 * 100)
```

## 💾 데이터베이스 구조

### 테이블 1: keyword_analysis
```sql
- id: 기본 키
- keyword: 키워드
- portal: 포털명 (Google/Naver/Daum/YouTube)
- search_volume: 추정 검색량
- trend: 트렌드 (rising/stable/declining)
- competition: 경쟁도 (Low/Medium/High)
- timestamp: 분석 시간
```

### 테이블 2: keyword_recommendations
```sql
- id: 기본 키
- keyword: 원본 키워드
- recommendation: 추천 키워드
- score: 점수 (0-100)
- category: 카테고리
- timestamp: 생성 시간
```

### 테이블 3: trend_data
```sql
- id: 기본 키
- keyword: 키워드
- date: 날짜
- search_volume: 검색량
- interest_level: 관심도
- portal: 포털명
- timestamp: 기록 시간
```

## 🎓 사용 예시

### 예시 1: 새 콘텐츠 주제 발굴
1. "인공지능" → Tab 4 → 실시간 추천
2. AI 관련 추천 키워드 20개 확인
3. "기계학습" 경쟁사 분석 → Tab 4 → 경쟁사 분석
4. 발견된 기회 키워드로 콘텐츠 계획

### 예시 2: 콘텐츠 최적화 시기 결정
1. "크리스마스" → Tab 5 → 계절성 감지
2. 피크 월: 11-12월 확인
3. 최고의 요일: 금요일 확인
4. 11월 금요일에 콘텐츠 발행 계획

### 예시 3: 포트폴리오 키워드 선정
1. "데이터분석" → Tab 2 → 단일 키워드 분석
2. 4개 포털 검색량 비교 (Naver 최고)
3. → Tab 4 → 숏/롱테일 분석
4. 숏테일로 브랜딩, 롱테일로 트래픽 수집
